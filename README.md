# Ejemplo de context engineering utilizando SPEC

##Contenido del proyecto
### agents.md
**Configura los roles, las capacidades y los permisos de las IAs que trabajan en el repositorio.**
### constitucion.md
**Define los principios innegociables, el tono de la comunicacion y las directrices eticas o de seguridad.**
### architecture.md
**Describe la estructura del sistema, el flujo de datos, la integracion de servicios y el stack tecnologico.**
### decision.md
**Registrar el porqué de las decisiones tomadas en el pasado para evitar ciclos infinitos de cambio.**
### spec.md
**Define que problemas resuelve la aplicacion y que funcionalidades debe tener. Es el puente entre el negocio y el codigo.**
